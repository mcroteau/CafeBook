Cafe

A Simple J2ee Project

This will walk you through step by step creating a simple blog application running locally on your development machine running h2 and Jetty. We will not dive too far into details, rather give you a 10,000 foot view of J2ee architecture.

What is J2ee

J2ee is short for Java Web Application Development. It is enterprise ready! It is modular, secular and beautiful! We hope you enjoy it as much as we do. Without further adoo, let's go further... and build our Cafe

We hope you are either running Linux, Windows with GitBash or Mac.

First things first...

We need to install OpenJdk and Maven. Installing simply means downloading the binaries and adding it to your system path. Here is how to do this.

Browse to 

https://adoptopenjdk.net/
and select an OpenJdk download. Download then open up your 
~/.bashrc
and add the following lines:

export JAVA_HOME=/{{location of java binary}}
export PATH=$PATH:$JAVA_HOME/bin

Next we need to add Maven. Maven is our build tool, it will find & download project dependencies for us and compile the project when deploying or running locally.

export MAVEN_HOME=/{{location of maven binary}}
export PATH=$PATH:$MAVEN_HOME/bin

Our final .bashrc should look something like:

#!/bin/bash

export JAVA_HOME=/Library/Java/JavaVirtualMachines/adoptopenjdk-8-openj9.jdk/Contents/Home
export PATH=$PATH:$JAVA_HOME/bin

export MAVEN_HOME=/Users/mcroteau/Development/Tools/apache-maven-3.6.3
export PATH=$PATH:$MAVEN_HOME/bin

Now lets "Source" which means compile our .bashrc so that java commands and maven commands are available!

source ~/.bashrc

We should now be able to type Java and Maven commands!

java -version

Should output something like the following

Eclipse OpenJ9 VM (build openj9-0.21.0, JRE 1.8.0 Mac OS X amd64-64-Bit Compressed References 20200728_646 (JIT enabled, AOT enabled)

Congratulations!

You are on your way... let's get started.

We need to create our project structure

First let create our project... open up a command prompt and type:

mkdir Cafe

then cd into your new Cafe directory

cd Cafe

Now we will create step by step each directory or folder that will hold supporting Servlets, Data Access Objects and Models.

Your project structure should look like:

Cafe/src/main/java/xyz/cafe/access
Cafe/src/main/java/xyz/cafe/common
Cafe/src/main/java/xyz/cafe/dao
Cafe/src/main/java/xyz/cafe/model
Cafe/src/main/java/xyz/cafe/startup
Cafe/src/main/java/xyz/cafe/web/post
Cafe/src/main/webapp/jsp/post
Cafe/src/main/WEB-INF

First lets create our directory to hold common Java files.

mkdir -p src/main/java/xyz/cafe/common

Let's do the same for Data Access Objects otherwise known as DAO

mkdir -p src/main/java/xyz/cafe/dao

Models or Domains

mkdir -p src/main/java/xyz/cafe/models

Startup

mkdir -p src/main/java/xyz/cafe/startup

Web or Controllers

mkdir -p src/main/java/xyz/cafe/web

If you are developer already then you are probably familiar with this project setup. If not, this is the cleanest most awesome way to configure your project. The pattern is called MVC which stands for Model View Controller. We will get to our views in a second. We already have web for controllers, and model for our models.
